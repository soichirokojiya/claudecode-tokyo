## Code Review機能とは

2026年3月9日、Claude Codeに**コードレビュー機能**がリサーチプレビューとして追加されました。

GitHub上のプルリクエスト（PR）に対して、複数のAIエージェントが並行してコードを分析し、バグ・セキュリティ脆弱性・エッジケースなどを検出してくれます。

## 使い方

### CLIから使う場合
```bash
claude review --pr 123
```

### GitHubから使う場合
PRのコメントに以下を入力するだけです。
```
@claude review
```

## 何を検出してくれる？

- **バグ**: ロジックエラー、null参照、境界条件の漏れ
- **セキュリティ脆弱性**: SQLインジェクション、XSS、認証バイパス
- **エッジケース**: 空配列、大量データ、並行処理の問題
- **回帰リスク**: 既存機能への影響

単なるスタイルチェックではなく、「このコードは本番環境で問題を起こす可能性があるか」という観点でレビューしてくれます。

## 気になるコスト

Code Review機能の平均コストは**1回あたり$15〜$25**です。

PRの規模（変更行数、ファイル数）によって大きく変わります。実際に「2つのPRのレビューで$100を超えた」という報告もあります。

### コストを抑えるには
- 小さなPRに分割する
- レビュー対象のファイルを限定する
- 重要なPRのみに使用する

## 対応プラン

- **Team / Enterprise プラン**: 利用可能（ベータ）
- **Pro / Max プラン**: 個人のCLIから利用可能
- **API**: 従量課金で利用可能

## 人間のレビューは不要になる？

結論から言うと、**まだ完全には代替できません**。

AIレビューが得意なのは「機械的に検出できるバグやパターン」です。ビジネスロジックの妥当性や、チームの方針との整合性などは、引き続き人間のレビューが必要です。

ただし、「見落としがちな技術的問題」をAIが先に検出してくれることで、人間はより本質的なレビューに集中できるようになります。

## まとめ

Code Review機能は非常に強力ですが、コストには注意が必要です。まずは重要度の高いPRで試してみて、チームの運用に合うか判断するのがおすすめです。
